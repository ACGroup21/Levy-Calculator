// the thesis

How we make it futuristic AND serious

The premium B2B sites (Linear, Stripe, Vercel) earn trust through restraint, not flash. The formula: a calm, high-contrast canvas + exactly one jaw-dropping interaction — and we make that interaction substantive, not decorative.

◐Restraint = serious

High contrast, generous whitespace, one accent colour (cyan), tight geometric type, obsessive consistency across every page.

✦One big moment = futuristic

A single high-craft live interaction in the hero. Futurism from depth, not from decorating every corner.

⚡Real tools = your edge

Stripe animates a gradient because it can't show payments. You can show levy money actually moving — for real.

// the big dynamic feature

Front-page hero — three concepts

Recommendation: build A as the hero, on a B-style ambient background, with C as the methodology section below. One coherent system.

RECOMMENDED
CONCEPT A

Watch your levy work

A live mini-tool: visitor sets their pay bill → levy pot resolves → particles stream from the pot into outcomes. Real math, real flow.

Role: The hero. Funnels straight into the calculator.
CONCEPT B

Living constellation

A drifting particle network that resolves into the "ac" mark — pure brand atmosphere via WebGL/canvas.

Role: The ambient background layer behind A.
CONCEPT C

The loop

Goal → Map → Fund → Report animates as you scroll, each stage drawing itself in. Scrollytelling.

Role: Methodology section under the hero.
References: Stripe — flowing gradient Linear — consistency Vercel — blueprint grid Three.js heroes

// build plan

Phases — shipped in slices

You see something real at every step. Nothing goes live until the domain cutover in the final phase.

Phase 0 — Brand & design system started

Logos integrated, palette & tokens, component set, clickable homepage mockup to approve the look. ← this map + the live prototype are here.

Phase 1 — Homepage shell layout

Full static homepage with real layout and placeholder hero.

Phase 2 — The hero centrepiece

Concept A built and tuned — the live levy-flow interaction.

Phase 3 — Tools hub integrate

Wire in the calculator + JD Matcher as the self-serve centre.

Phase 4 — Remaining pages content

Approach, For Employers, About, Contact.

Phase 5 — Polish craft

Copywriting, responsive, performance, accessibility, SEO/meta, image optimisation.

Phase 6 — Launch cutover

DNS from Wix → GitHub Pages. ~10 minutes, zero downtime.
